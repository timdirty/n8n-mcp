# STEAM 教育比賽資訊蒐集系統架構

## 系統概述

完整的自動化系統，用於蒐集、處理、分類和推送 STEAM 教育比賽資訊。

## 架構設計

### 1. 資料來源層（5 個節點）

#### RSS Feed 節點 × 3
- **教育部 RSS**：教育部官方公告
- **縣市教育局 RSS**：各縣市教育局公告
- **STEAM 平台 RSS**：專業 STEAM 教育平台

#### HTTP Request 節點 × 2
- **API 資料源**：第三方教育 API
- **網頁爬蟲**：特定網站資訊抓取

### 2. 資料處理層（8 個節點）

#### Function 節點：資料正規化
- 統一資料格式（標題、描述、URL、日期、來源）
- 處理不同來源的資料結構差異
- 資料清洗和驗證

#### Function 節點：關鍵字過濾
- STEAM 相關關鍵字匹配
- 排除無關資訊
- 優先級評分

#### Function 節點：5 層去重機制
1. **Hash 去重**：MD5 雜湊值比對
2. **URL 去重**：網址唯一性檢查
3. **標題相似度**：Levenshtein distance < 0.85
4. **時間窗口**：7 天內重複檢查
5. **內容相似度**：TF-IDF + Cosine similarity < 0.90

#### OpenAI 節點：AI 分類
- 使用 GPT-4o-mini 模型
- 分類為：科學(S)、技術(T)、工程(E)、藝術(A)、數學(M)
- 多標籤分類支援

#### OpenAI 節點：AI 摘要生成
- 生成 100-150 字摘要
- 提取關鍵資訊（時間、地點、對象、獎項）
- 繁體中文輸出

#### Function 節點：Flex Message 格式化
- LINE Flex Message 結構
- 圖片、標題、摘要、按鈕
- 響應式設計

#### Function 節點：資料驗證與錯誤處理
- 必填欄位檢查
- 資料類型驗證
- 錯誤日誌記錄

#### Switch 節點：依分類路由
- 根據 STEAM 分類分流
- 多路徑輸出

### 3. 資料儲存層（3 個節點）

#### Google Sheets 節點：主資料表
- 所有比賽資訊
- 完整欄位記錄
- 自動時間戳記

#### Google Sheets 節點：分類資料表
- 依 STEAM 類別分表
- 便於統計分析

#### Notion 節點：同步到 Notion
- Notion 資料庫整合
- 支援標籤和關聯
- 團隊協作

### 4. 通知層（3 個節點）

#### LINE 節點：主群組通知
- 發送到主要 LINE 群組
- Flex Message 格式
- 包含所有分類

#### LINE 節點：分類群組通知
- 依 STEAM 類別發送到不同群組
- 精準推送

#### HTTP Request 節點：Webhook 通知
- 通知後台管理系統
- 觸發其他自動化流程

### 5. 管理與監控層（6 個節點）

#### Schedule Trigger 節點
- 定時執行：每 6 小時
- Cron 表達式：`0 */6 * * *`

#### Webhook 節點
- 手動觸發端點
- 支援參數傳遞

#### Error Trigger 節點
- 捕獲工作流程錯誤
- 自動重試機制

#### Function 節點：執行日誌
- 記錄執行時間
- 統計處理數量
- 錯誤追蹤

#### HTTP Request 節點：健康檢查
- 定期檢查系統狀態
- 監控 API 可用性

#### Function 節點：統計資料生成
- 每日/每週/每月統計
- 分類分布分析
- 趨勢報告

### 6. 測試層（3 個節點）

#### Manual Trigger 節點
- 手動測試觸發
- 開發調試用

#### Function 節點：測試資料生成
- 生成模擬資料
- 測試各種情境

#### HTTP Request 節點：測試結果驗證
- 驗證輸出正確性
- 自動化測試

## 資料流程圖

```
[資料來源] → [正規化] → [過濾] → [去重] → [AI分類] → [AI摘要] → [格式化]
                                                                    ↓
[測試驗證] ← [統計分析] ← [日誌記錄] ← [錯誤處理] ← [路由分流]
                                                                    ↓
                                        [Google Sheets] ← [資料驗證]
                                        [Notion 同步]
                                        [LINE 通知]
                                        [Webhook 通知]
```

## 技術規格

### 節點總數
- 總計：25-30 個節點
- 資料來源：5 個
- 資料處理：8 個
- 資料儲存：3 個
- 通知：3 個
- 管理監控：6 個
- 測試：3 個

### 性能指標
- 處理速度：< 5 秒/筆
- 去重準確率：> 95%
- AI 分類準確率：> 90%
- 系統可用性：> 99%

### 擴展性
- 支援新增資料來源
- 支援自定義分類規則
- 支援多語言處理
- 支援自定義通知渠道
